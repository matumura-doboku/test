・pyodideを経由したAPIデータの取得とシステムへの合流機能の作成
・構想
データ加工タブからpythonを実行してデータをjson,geojsonとして取得できる機能
jsonデータはアイコンとして表示できるようにし、色やメッシュ内の個数で傾向を出せるようにする
Geojsonはメッシュに重ねてこれまで通りの表示方法になる
・手順
１データ加工からデータを取得できるようにUI処理を整備する（APIキーを安全に扱い、国交省のルールに従った通信を行います。ヘッダーへのセット: APIキーをHTTPヘッダー（Authorization等）に正しくセットする処理を書きます HTTPS通信の維持: ブラウザ（Pyodide）が実行する通信は自動的にTLS/SSLで暗号化されますが、プログラム内でリクエスト先が https:// で始まっていることを厳格に管理します 。）
APIキーは使用者が自分のキーを入力できるようにUIを作る。地域を選択できるようにしておく。
２データをシステムで使えるように加工する座標系や、形式を合わせてシステムで使えるようにする。
３できたデータをアップしてシステムで使えるようにデータアップ機能を作る
・使用するAPI
分野,主要API・プラットフォーム,取得できる主なデータ,通信方式・形式
インフラ老朽化,xROAD（全国道路施設点検DB）,橋梁・トンネルの健全度（Ⅰ〜Ⅳ）、点検調書,REST (JSON)
防災・安全,国土数値情報 (KSJ),浸水想定区域、土砂災害警戒区域,ArcGIS Feature Service
都市・経済,RESAS API / xROAD,地価公示、道路交通センサス（交通量）,REST / GraphQL
交通・人流,公共交通オープンデータ(ODPT),バス停・駅座標、GTFS（時刻表・動態）,Web API (JSON/GTFS)
